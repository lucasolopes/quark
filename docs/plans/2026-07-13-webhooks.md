# Signed outgoing webhooks — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Push HMAC-signed HTTP events (`link.created/updated/deleted/expired/clicked`) to operator-configured endpoints, following the Standard Webhooks spec.

**Architecture:** Subscription config persisted via `Store` (LMDB + Postgres). Events flow through a bounded `mpsc` channel to a background worker that mirrors `analytics::spawn_worker`; the worker matches active subscriptions, signs per Standard Webhooks, and POSTs with retry (best-effort, in-memory). The redirect hot path emits `clicked`/`expired` only behind a cached `AtomicBool`, then a non-blocking `try_send`.

**Tech Stack:** Rust (axum, tokio mpsc, reqwest [new dep], hmac+sha2, base64, serde), React 19 + Vite + Vitest.

## Global Constraints

- Redirect hot path pays at most one `Ordering::Relaxed` atomic load when no `clicked`/`expired` subscription exists; never a synchronous POST.
- Signing follows Standard Webhooks exactly: headers `webhook-id`, `webhook-timestamp`, `webhook-signature: v1,<base64(HMAC-SHA256(secret_bytes, "{id}.{ts}.{body}"))>`; secret is `whsec_<base64>`; the signed `body` is the exact bytes sent.
- SSRF: every destination URL validated with `abuse::is_internal_host` at create time AND delivery time; only `http`/`https`.
- Subscription config persisted via `Store` (LMDB + Postgres); delivery is in-memory best-effort. Fail-open on channel-full and delivery error.
- All code English; UI strings via i18n (EN + PT-BR); docs EN + `PT_BR`.
- Rust tests run with `-j1` / `CARGO_BUILD_JOBS=1` (parallel-build ICE in this env); Postgres-gated tests read `QUARK_TEST_DATABASE_URL` and skip cleanly when unset. Stop any running `quark.exe` before `cargo build/bench` (it locks the release exe).
- Stay on `feat/webhooks`; do NOT merge to main.

```rust
pub struct WebhookDispatcher {
    tx: tokio::sync::mpsc::Sender<WebhookEvent>,
    pub clicked_subscribed: std::sync::Arc<std::sync::atomic::AtomicBool>,
    pub expired_subscribed: std::sync::Arc<std::sync::atomic::AtomicBool>,
}
impl WebhookDispatcher {
    pub fn emit(&self, ev: WebhookEvent);          // non-blocking try_send, fail-open (drop + log)
}
pub fn spawn_webhook_worker(
    rx: tokio::sync::mpsc::Receiver<WebhookEvent>,
    store: std::sync::Arc<dyn Store>,
    flags: WebhookFlags,               // the two AtomicBools, refreshed by the worker
) -> tokio::task::JoinHandle<()>;

pub const DELIVERY_ATTEMPTS: u32 = 3;
pub const DELIVERY_TIMEOUT_SECS: u64 = 5;
```
